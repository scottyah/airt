## Local Development

### Quick Start (Recommended)

```bash
# One command to install and run
./local-dev.sh

# Open browser
open http://localhost:3000
```

The script automatically installs dependencies if needed and starts the server with hot reload.

### Manual Setup

```bash
# Install dependencies
cd server && npm install

# Run server with hot reload
cd ..
npm run dev

# Or run without hot reload
npm start
```

## Docker Build & Deployment

### Prerequisites

- Docker installed
- kubectl configured for your cluster
- Access to Harbor registry: `harbor.scottyah.com`
- Cluster has `harborcred` secret configured

### Build and Deploy

```bash
# Build and deploy in one command
./ship.sh

# Or build only
./ship.sh --build-only

# Or deploy only
./ship.sh --deploy-only
```

This will:
1. Build multi-stage Docker image
2. Tag with version, timestamp, and `latest`
3. Push to Harbor registry at `harbor.scottyah.com/scottyah/airt`
4. Create `airt` namespace (if not exists)
5. Deploy service, deployment, and ingress
6. Restart deployment with latest image
7. Wait for rollout to complete

The site will be available at: **https://airt.scottyah.com**

## Adding New Exhibits

Each exhibit is self-contained in its own directory:

```
public/js/exhibits/{exhibit-id}/
â”œâ”€â”€ index.js       # Exhibit class
â””â”€â”€ config.json    # Metadata
```

### 1. Create exhibit directory

```bash
mkdir -p public/js/exhibits/my-exhibit
```

### 2. Create `config.json`

```json
{
  "id": "my-exhibit",
  "title": "My Amazing Exhibit",
  "description": "Description of what this exhibit does",
  "category": "fractal|generative|3d",
  "library": "canvas|p5|three",
  "thumbnail": "/assets/thumbnails/my-exhibit.jpg",
  "tags": ["tag1", "tag2"],
  "instructions": [
    "Instruction 1",
    "Instruction 2"
  ]
}
```

### 3. Create `index.js`

```javascript
export default class MyExhibit {
  constructor(container, config) {
    this.container = container;
    this.config = config;
  }

  async init() {
    // Setup canvas, initialize libraries
  }

  start() {
    // Begin animation loop
  }

  stop() {
    // Stop animation
  }

  resize() {
    // Handle window resize
  }

  reset() {
    // Reset to initial state
  }

  togglePause() {
    // Pause/resume animation
  }

  destroy() {
    // Cleanup
  }
}
```

### 4. Register exhibit

Add to `public/js/exhibits/registry.json`:

```json
{
  "exhibits": [
    {
      "id": "my-exhibit",
      "enabled": true
    }
  ]
}
```

### 5. Create thumbnail

Add screenshot to `assets/thumbnails/my-exhibit.jpg`

### 6. Rebuild and deploy

```bash
./ship.sh
```

## Keyboard Shortcuts

- **ESC** - Return to gallery
- **Space** - Pause/resume animation (if supported)
- **R** - Reset exhibit to initial state

## Performance

- Target: 60 FPS for all exhibits
- Canvas rendering uses `devicePixelRatio` for crisp displays
- `requestAnimationFrame` for smooth animations
- Web Workers for compute-intensive fractals
- Proper cleanup to prevent memory leaks

- Modern browsers with WebGL 2.0 support
- Chrome, Firefox, Safari, Edge (latest versions)
- Mobile browsers (iOS Safari, Chrome Mobile)
